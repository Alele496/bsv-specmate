# P0200 — BVI schedule 缺少方法间的调度注解

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 1 captures

## 现象

```
Warning: "UartRxWrapper.bsv", line 12, column 10: (P0200)
  No scheduling annotation given between method rx_data and methods (rx_done, rx_frame_err)
```

## 原因

BVI import 模块中有多个 value method，两两之间需要 CF（conflict-free）调度注解。BSC 编译器要求逐对声明，**不支持分组语法**。

## 解决方案

**逐对声明 schedule 注解，不使用分组：**

```bsv
// 错误 — 分组语法不被接受
schedule rx_data CF (rx_data, rx_done, rx_frame_err);

// 正确 — 必须逐对声明
schedule rx_data      CF rx_data;
schedule rx_data      CF rx_done;
schedule rx_data      CF rx_frame_err;
schedule rx_done      CF rx_data;
schedule rx_done      CF rx_done;
schedule rx_done      CF rx_frame_err;
schedule rx_frame_err CF rx_data;
schedule rx_frame_err CF rx_done;
schedule rx_frame_err CF rx_frame_err;
```

## 规则

> BVI schedule 声明不支持 `schedule A CF (B, C, D)` 分组语法。必须展开为逐对 `schedule A CF B` 形式。N 个 value method 需要 N x N 条 schedule 声明（包括自引用 `schedule A CF A`）。
