# P0005 — 标识符与 BSV/SV 保留字冲突 (×6)

P0005 是 BSC 的通用语法解析错误码，对应多种子场景。以下按触发位置分类。

## 场景 1：SV 保留字作为标识符名

**现象**

```
Error: "...", line ..., column ...: (P0005)
  Unexpected ... keyword `xxx'; expected ...
```

| 冲突词 | 类型 | 出现位置 | 改为 |
|--------|------|----------|------|
| `action` | BSV 关键字 | struct 字段名 | `act` |
| `bit` | SV 关键字 | 方法参数名 | `val` |
| `byte` | SV 关键字 | 局部变量 `let` | `b` |
| `priority` | SV 3.0 关键字 | 寄存器名 | `pri_idx` / `pr` |
| 大写 `ACT_*` | 大写=类型名语义 | package 常量 | `actWrCh` 等驼峰 |

**原因**

BSV 编译时检查包含 BSV 关键字和 SystemVerilog 关键字的冲突。SV 保留字参考（不完整列表）：`action`, `bit`, `byte`, `reg`, `wire`, `module`, `input`, `output`, `inout`, `assign`, `always`, `initial`, `posedge`, `negedge`, `case`, `default`, `endcase`, `begin`, `end`, `function`, `task`, `class`, `interface`, `package`, `import`, `parameter`, `localparam`, `priority`, `specify`, `primitive`, `buf`, `bufif0`, `bufif1`, `not`, `and`, `nand`, `or`, `nor`, `xor`, `xnor`。DMA 实验中 `buf` 变量名冲突导致 T0011 类错误，现已在 check_style 检测中覆盖。

## 场景 2：`always_enabled` suffix 在 module 内 method 实现上（位置错误，新增）

**现象**

```
Error: "TopController.bsv", line 151, column 44: (P0005)
  Unexpected identifier `always_enabled'; expected BSV 3.8 keyword `provisos',
  method condition, `;', or `='
```

**原因**

BSC 的 BSV 语法中，module 内 method 实现后面只能跟三种东西：provisos 声明（类型约束）、method guard 条件（如 `if (cond)`）、分号 `;`（Action 方法）或等号 `=`（Value 方法）。`always_enabled` suffix 关键字在这里不是合法 token——它与 provisos 关键字不是同一语法类别。这是 BSC 解析器的语法位置约束，不是保留字冲突。注意：同一位置的 `(* always_enabled *)` pragma 形式会触发 P0022 错误。两种形式在 module 内 method 实现上均不合法。

**解决**

去掉 method 实现后面的 `always_enabled` suffix，保持普通形式（无显式 guard）。消除 G0015 的正确做法是在 interface 声明层加 `(* always_enabled *)` pragma——module 内 method 实现不需要任何标记，没有显式 guard 的 method 默认每拍可调用，编译器通过 interface 层的契约推导 BVI 子模块的调用保证。

错误示范：
```bsv
method Action uart_rx_pin(Bit#(1) val) always_enabled;  // ❌ P0005
```

正确写法：
```bsv
// Interface 声明（唯一需要加 pragma 的地方）
(* always_enabled *)
method Action uart_rx_pin(Bit#(1) val);

// Module 内实现（普通形式，不加任何标记）
method Action uart_rx_pin(Bit#(1) val);
    uart_rx.uart_rx_in(val);
endmethod
```

> **规则**: 所有标识符一律用小写或驼峰，避开 SV 保留字。module 内 method 实现后面只能跟 provisos、guard 条件、`;` 或 `=`——不要加 `always_enabled` suffix 关键字（消除 G0015 请用 interface 声明层的 `(* always_enabled *)` pragma）。package 常量也用小写。
