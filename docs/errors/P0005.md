# P0005 — 标识符与 BSV/SV 保留字冲突 (×6)

**bsc 输出**：
```
Error: "...", line ..., column ...: (P0005)
  Unexpected ... keyword `xxx'; expected ...
```

**根本原因**：BSV 编译时检查包含 BSV 关键字和 SystemVerilog 关键字的冲突。

| 冲突词 | 类型 | 出现位置 | 改为 |
|--------|------|----------|------|
| `action` | BSV 关键字 | struct 字段名 | `act` |
| `bit` | SV 关键字 | 方法参数名 | `val` |
| `byte` | SV 关键字 | 局部变量 `let` | `b` |
| `priority` | SV 3.0 关键字 | 寄存器名 | `pri_idx` / `pr` |
| 大写 `ACT_*` | 大写=类型名语义 | package 常量 | `actWrCh` 等驼峰 |

**SV 保留字参考**（不完整列表，详见 SV 标准）：
`action`, `bit`, `byte`, `reg`, `wire`, `module`, `input`, `output`, `inout`, `assign`, `always`, `initial`, `posedge`, `negedge`, `case`, `default`, `endcase`, `begin`, `end`, `function`, `task`, `class`, `interface`, `package`, `import`, `parameter`, `localparam`, `priority`, `specify`, `primitive`, `buf`, `bufif0`, `bufif1`, `not`, `and`, `nand`, `or`, `nor`, `xor`, `xnor`

> 新增：`buf`, `bufif0`, `bufif1` (SV gate primitives), `not`, `and`, `nand`, `or`, `nor`, `xor`, `xnor` (SV gate primitives) — DMA 实验中 `buf` 变量名冲突导致 T0011 类错误，现已在 check_style 检测中覆盖。

> **规则**: 所有标识符一律用小写或驼峰，避开 SV 保留字。package 常量也用小写。
