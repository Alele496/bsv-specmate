# G0010 — 跨 rule 方法调用冲突 (×3)

**bsc 输出**（警告）：
```
Warning: "...", line 56, column 8: (G0010)
  Rule "rl_uart_tx_send" was treated as more urgent than "utx_rl_tx".
```

---

## 子情况 1 — PulseWire+Reg 传数据导致丢字节

**原因**：`PulseWire` 将事件在同 cycle 传给 rule，但与之搭配的 `Reg` 在当前 cycle 不可见 → 第一个字节丢失。

**解决**：改用 `FIFOF` 传递数据。

```bsv
FIFOF#(Bit#(8)) tx_fifo <- mkFIFOF();

method Action tx_start(Bit#(8) data) if (tx_fifo.notFull);
    tx_fifo.enq(data);
endmethod

rule rl_tx;
    if (tx_idle && tx_fifo.notEmpty) begin
        shift_reg <= {1'b1, tx_fifo.first, 1'b0};
        tx_fifo.deq();
    end
endrule
```

## 子情况 2 — 跨模块调度需显式标注 urgency

**原因**：外部 module 的 method 写入寄存器，本 module 内部 rule 读取同一寄存器 → BSV 自动推断 urgency 但报 warning。

**解决**：加 urgency 注解。

```bsv
(* descending_urgency = "rl_dac_consume, dac_rl_spi" *)
```

> **规则**: 跨 rule 触发+数据用 FIFOF。跨模块 method/rule 互斥需显式标注 urgency。
