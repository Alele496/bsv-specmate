# T0132 — Sized Literal 值超位宽 (x2)

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 2 captures

## 子情况 1：字面量值超出声明位宽

**bsc 输出**：
```
Error: "Top.bsv", line 45, column 25: (T0132)
  Literal 200 is not a valid Bit#(7).
  Bit#(7) has maximum value 127.
```

**原因**：

`Bit#(n)` 寄存器只能存储 0 到 2^n-1 的值（无符号）或 -2^(n-1) 到 2^(n-1)-1（有符号）。赋值的字面量超出了这个范围。

常见触发：扩大某值后，接收该值的寄存器位宽未同步扩大。

**解决**：

扩大寄存器的位宽。

```bsv
// 错误
Reg#(Bit#(7)) counter <- mkReg(0);
rule rl_count;
    // counter 最大只能存 127，200 超出了
    counter <= 200;              // ❌ T0132
endrule

// 正确 — 扩大到位宽 8
Reg#(Bit#(8)) counter <- mkReg(0);
rule rl_count;
    counter <= 200;              // ✅ Bit#(8) 最大 255
endrule
```

## 子情况 2：零位宽字面量

**bsc 输出**：
```
Warning: "Top.bsv", line 72, column 30: (T0132)
  Zero-width literal: Bit#(0) cannot hold any value.
```

**原因**：

`Bit#(0)` 声明了 0 位宽的寄存器/字面量——这是无效的，0 比特不能存储任何值。

**解决**：

使用至少 `Bit#(1)` 或用 `?` 表示 don't-care。

```bsv
// 错误 — 0 位宽无意义
Reg#(Bit#(0)) unused <- mkReg(?);  // ❌ Bit#(0)

// 正确 — 用 Bit#(1) 或直接移除该寄存器
Reg#(Bit#(1)) unused <- mkReg(0);
```

## 规则

> Sized literal 的值必须在位宽范围内（无符号：[0, 2^n-1]）。扩大计算逻辑时，同步扩宽所有相关寄存器的位宽。位宽至少 1 位。
