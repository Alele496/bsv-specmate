# G0002 — RegFile 读端口超限

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 1 captures

## 现象

```
Error: "Top.bsv", line 45, column 12: (G0002)
  Submodule `rf' of type `RegFile' has too many read methods called in a single clock cycle.
  Each RegFile supports at most 5 read ports per cycle.
```

## 原因

BSV 中 `mkRegFile` / `mkRegFileFull` 硬件原语每个 cycle 最多支持 5 个读端口。同一 cycle 内调用超过 5 次 `rf.sub()`（或对应的 read method）时，综合工具无法映射到真实的 BRAM 硬件。

常见触发场景：
- 仲裁器设计：同一 cycle 内多个请求者同时查表
- 多级流水线：同一 cycle 中多个 stage 都需要读 BRAM
- 宽接口模块：外部 method 调用触发多个内部 sub() 调用

## 解决方案

**方案 1（推荐）：拆分到多个 cycle**

```bsv
// 错误 — 同一 rule 内 6+ 次 rf.sub()
rule rl_process;
    let v0 = rf.sub(addr0);
    let v1 = rf.sub(addr1);
    let v2 = rf.sub(addr2);
    let v3 = rf.sub(addr3);
    let v4 = rf.sub(addr4);
    let v5 = rf.sub(addr5);  // G0002 — 第 6 个读端口
endrule

// 正确 — 拆为两个 cycle
rule rl_process_phase1 (state == PHASE1);
    let v0 = rf.sub(addr0);
    let v1 = rf.sub(addr1);
    let v2 = rf.sub(addr2);
    // 存入中间寄存器，下一 cycle 继续
    state <= PHASE2;
endrule
rule rl_process_phase2 (state == PHASE2);
    let v3 = rf.sub(addr3);
    let v4 = rf.sub(addr4);
    let v5 = rf.sub(addr5);
    state <= PHASE1;
endrule
```

**方案 2：用 BRAM 替代 RegFile**

若确实需要超过 5 读端口，考虑使用 `BRAMCore` 或 `BRAM`（更灵活但需手动管理读写端口时序）：

```bsv
BRAM_Configure cfg = defaultValue;
cfg.memorySize = 256;
BRAM1Port#(Bit#(8), Bit#(32)) bram <- mkBRAM1Server(cfg);
// BRAMCore 支持流水线读取，可复用 read 端口
```

## 规则

> RegFile 硬件限制 5 读端口/cycle。超出的设计拆到多个 cycle，或用 BRAM/BRAMCore 替代。
