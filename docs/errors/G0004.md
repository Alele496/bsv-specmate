# G0004 — Rule 内并行写入同一寄存器 (×1)

**bsc 输出**：
```
Error: "...", line 77, column 10: (G0004)
  Rule `RL_dac_rl_spi' uses methods that conflict in parallel:
    dac_busy_r.write(...)
  and
    dac_busy_r.write(...)
```

**原因**：同一 rule 内对同一寄存器编写了两次写入。BSV 编译器必须静态保证每个寄存器在单一 rule 内最多被写入一次。`case` 的 `default` 分支即使不执行，若写入与 case 内分支相同的寄存器，也会触发。

```bsv
// 错误
rule rl_spi;
    busy_r <= True;        // 写入 #1
    case (cmd_type)
        WRITE: begin
            busy_r <= False; // 写入 #2 → G0004
        end
        default: begin
            busy_r <= False; // 写入 #3 → 同样冲突
        end
    endcase
endrule

// 正确：删除 case 前的写入或重构逻辑
rule rl_spi;
    case (cmd_type)
        WRITE: begin busy_r <= False; end
    endcase
endrule
```

> **规则**: 同一 rule 内每个寄存器只能在一个无条件路径上被写入。自枚举类型不需要 default 分支保护。
