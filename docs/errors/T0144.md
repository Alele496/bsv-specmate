# T0144 — Union tagged 构造/访问错误 (x2)

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 2 captures

## 子情况 1：tagged 构造带数据的 tag 未传参

**bsc 输出**：
```
Error: "Top.bsv", line 55, column 22: (T0144)
  Tag "Valid" of union type "Result" expects 1 data argument, but 0 were given.
```

**原因**：

Union 的某个 tag 声明了附带数据（如 `Valid Bit#(32)`），但构造时只写了 tag 名没有传数据参数。

**解决**：

`tagged Valid data_val` 格式——tag 名后跟数据值。

```bsv
// union 定义
typedef union tagged {
    void     Invalid;
    Bit#(32) Valid;
    void     Overflow;
} Result;

// 错误
Result r = tagged Valid;          // ❌ Valid 需要 Bit#(32) 参数

// 正确
Result r = tagged Valid 32'hAABB; // ✅ 数据值必须提供
Result r2 = tagged Invalid;       // ✅ Invalid 是 void tag，不需参数
```

## 子情况 2：用 .field 方式访问 union 字段

**bsc 输出**：
```
Error: "Top.bsv", line 68, column 15: (T0144)
  Union field access is not allowed. Use `case matches' to extract tagged values.
```

**原因**：

Union 字段不能像 struct 那样用 `.field` 直接访问——因为 tag 值在运行时才确定，编译器无法保证字段存在。

**解决**：

用 `case matches` 模式匹配。

```bsv
// 错误
Bit#(32) data = result.Valid;    // ❌ 不能 .field 访问 union

// 正确
Bit#(32) data = ?;
case (result) matches
    tagged Valid .d: data = d;   // .d 捕获 Valid tag 内的值
    tagged Invalid:  data = 0;
    tagged Overflow: data = 32'hFFFF;
endcase
```

## 规则

> Union 使用 `tagged Tagname value` 构造带数据的 tag，用 `case matches` 做模式匹配提取值。不能 `.field` 直接访问——union 的 tag 是运行时的，编译器无法静态确定。
