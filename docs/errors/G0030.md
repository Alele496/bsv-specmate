# G0030 — Urgency 属性形成循环依赖

## 现象

```
Error: "Top.bsv", line 22, column 5: (G0030)
  Attribute `descending_urgency' forms a cycle in the rule scheduling graph.
  Rule "rl_a" is more urgent than "rl_b" which is more urgent than "rl_a".
```

## 原因

`descending_urgency`、`execution_order` 等调度属性构成了循环依赖图。主要有两种触发形式：

1. **跨声明循环**：多行 `(* descending_urgency *)` 形成闭环
   ```bsv
   (* descending_urgency = "rl_a, rl_b" *)    // rl_a > rl_b
   (* descending_urgency = "rl_b, rl_c" *)    // rl_b > rl_c
   (* descending_urgency = "rl_c, rl_a" *)    // rl_c > rl_a → 形成循环 rl_a > rl_b > rl_c > rl_a
   ```

2. **单行自环**：同一规则在声明中重复引用
   ```bsv
   (* descending_urgency = "rl_a, rl_b, rl_a" *)  // rl_a 出现两次 → 自环
   ```

## 解决方案

**检出循环的规则链，移除形成闭环的那条依赖声明。**

```bsv
// 错误 — 形成 rl_a > rl_b > rl_c > rl_a 循环
(* descending_urgency = "rl_a, rl_b" *)
(* descending_urgency = "rl_b, rl_c" *)
(* descending_urgency = "rl_c, rl_a" *)    // ← 这条形成循环，删除它

// 正确 — 移除闭环
(* descending_urgency = "rl_a, rl_b" *)
(* descending_urgency = "rl_b, rl_c" *)
// rl_c 和 rl_a 之间由编译器自动推断调度顺序
```

**单行自环修复**：

```bsv
// 错误 — rl_a 在同一声明中重复
(* descending_urgency = "rl_a, rl_b, rl_a" *)

// 正确 — 移除重复
(* descending_urgency = "rl_a, rl_b" *)
```

## 规则

> urgency 声明的优先级关系必须构成有向无环图（DAG）。检查所有 `descending_urgency` / `execution_order` — 确保没有 A > B > ... > A 的路径。
