# T0061 — Bool 与 Bit 类型混淆 (×2)

**bsc 输出**（位操作符误用）：
```
Error: "...", line ..., column ...: (T0061)
  Bitwise operators are not defined for Boolean values. Consider using the
  Boolean operators && and || instead.
```

**bsc 输出**（Bool 用于位拼接 / Bit#(1) 赋值给 Bool — 报 T0020）：
```
Error: "Timer.bsv", line ..., column ...: (T0020)
  Type error — cannot use Bool in a Bit context.
```

**根本原因**：`Bool` 和 `Bit#(1)` 是 BSV 中两个不同的类型，不能混用。

## 常见错误模式

### 1. Bool 使用位操作符

```bsv
// 错误
Reg#(Bool) wave <- mkReg(False);
wave <= ~wave;     // Bool 不能用 ~

// 正确
wave <= !wave;
```

### 2. Bool 用于位拼接 ✨新增

```bsv
// 错误
Reg#(Bool)     enable      <- mkReg(False);
Reg#(Bool)     auto_reload <- mkReg(False);
return {30'd0, auto_reload, enable};  // Bool 不能拼接进 Bit

// 正确：改用 Bit#(1)
Reg#(Bit#(1))  enable      <- mkReg(0);
Reg#(Bit#(1))  auto_reload <- mkReg(0);
return {30'd0, auto_reload, enable};  // 合法：Bit#(1)+Bit#(1)→Bit#(2)
```

### 3. Bit#(1) 直接赋给 Bool 寄存器 ✨新增

```bsv
// 错误
Reg#(Bool) enable <- mkReg(False);
Reg#(Bit#(32)) wdata <- mkReg(0);
enable <= wdata[0];   // wdata[0] 是 Bit#(1)，不能赋给 Bool

// 正确
enable <= unpack(wdata[0]);  // 显式转换
// 或直接用 Bit#(1) 寄存器
```

## 类型对照

| 类型 | 操作符 | 位拼接 | 寄存器建议 |
|------|--------|--------|-----------|
| `Bool` | `!` `&&` `\|\|` `==` `!=` | ❌ 不能 | `Reg#(Bool)` **仅**用于纯逻辑信号 |
| `Bit#(1)` | `~` `&` `\|` `^` `==` `!=` | ✅ 可以 | `Reg#(Bit#(1))` 用于可拼接的控制信号 |

> **规则**: 控制信号如需位拼接或从 Bus 中提取，优先用 `Bit#(1)` 而非 `Bool`。纯逻辑条件（不会出现在总线中）用 `Bool`。
