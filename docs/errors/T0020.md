# T0020 — 操作符类型不匹配

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-20 | 来源: 知识条目

**bsc 输出**：
```
Error: "...", line X, column Y: (T0020)
  Type mismatch in operator application.
  Expected operand type for "~" does not match actual type "Bool".
```

**原因**：

BSV 严格区分位级操作符和逻辑操作符。`~`（按位 NOT）、`&`（按位 AND）、`|`（按位 OR）期望 `Bit#(n)` 类型操作数。`!`（逻辑 NOT）、`&&`（逻辑 AND）、`||`（逻辑 OR）期望 `Bool` 类型操作数。混用触发 T0020。

**解决**：

始终使用类型匹配的操作符。

```bsv
// 错误 — ~ 用于 Bool
Bool done = True;
Bool done_inv = ~done;  // T0020

// 正确 — ! 用于 Bool
Bool done = True;
Bool done_inv = !done;

// 错误 — ! 用于 Bit#(1)
Bit#(1) flag = 1'd1;
Bit#(1) flag_inv = !flag;  // T0020

// 正确 — ~ 用于 Bit#(1)
Bit#(1) flag = 1'd1;
Bit#(1) flag_inv = ~flag;
```

> **规则**: BSC 2025.07 的类型检查器已能捕获绝大多数操作符混用情况（! 对 Bit#(1)、~ 对 Bool 均报 T0020）。硬件信号用 Bit#(1)，纯逻辑判断用 Bool。
