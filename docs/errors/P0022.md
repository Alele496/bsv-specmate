# P0022 — Method 上不允许使用 attribute pragma

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 1 captures

## 现象

```
Error: "Top.bsv", line 42, column 10: (P0022)
  Attributes not allowed for method
```

## 原因

在 **module 内 method 实现**上使用了 `(* always_enabled *)` pragma 形式。BSC 编译器对 interface 声明和 module 实现的语法要求是不对称的，且两种方式在 module 内实现上**均不合法**（经实际编译验证）：

| 位置 | `(* always_enabled *)` pragma | `always_enabled` suffix 关键字 |
|------|------------------------------|-------------------------------|
| Interface method 声明 | 合法 | 合法 |
| Module 内 method 实现 | **P0022** | **P0005**（suffix 在此位置不是合法 token） |
| BVI import method | 合法 | 不适用 |

> `always_enabled` suffix 关键字在 module 内 method 实现上触发的是 P0005（不是 P0022），因为 BSC 的 BSV 语法中 method 实现后面只能跟 provisos 声明、method guard 条件、分号 `;`（Action 方法）或等号 `=`（Value 方法）。`always_enabled` 在这里不是合法 token。

## 解决方案

**消除 G0015 的正确方法：仅在 interface 声明上加 `(* always_enabled *)` pragma，module 内 method 实现保持普通形式即可。**

没有显式 guard 的 method 默认就是每拍可调用的——编译器通过 interface 层的契约推导 BVI 子模块 method 的调用保证。**不需要在 module 实现侧做任何标记。**

```bsv
// Interface 声明（pragma 形式——唯一需要加的地方）
(* always_enabled *)
method Action uart_rx_pin(Bit#(1) val);

// Module 内实现（普通形式——不加任何标记）
method Action uart_rx_pin(Bit#(1) val);
    uart_rx.uart_rx_in(val);
endmethod
```

P0022 的错误消息只说 "Attributes not allowed for method"，没有提示 module 内实现上 suffix 关键字也会触发另一个错误（P0005）。两者在 module 实现侧均不可用，治本方案是只在 interface 声明层加 pragma。

## 规则

> Module 内 method 实现上 **不能** 使用 `(* always_enabled *)` pragma（触发 P0022），也 **不能** 使用 `always_enabled` suffix 关键字（触发 P0005）。消除 G0015 的正确方法：仅在 interface 声明上加 `(* always_enabled *)` pragma，module 内 method 实现保持无标记的普通形式。
