# P0022 — Method 上不允许使用 attribute pragma

## 现象

```
Error: "Top.bsv", line 42, column 10: (P0022)
  Attributes not allowed for method
```

## 原因

在 **module 内 method 实现**上使用了 `(* always_enabled *)` pragma 形式。BSC 编译器对 interface 声明和 module 实现的语法要求是不对称的：

| 位置 | `(* always_enabled *)` pragma | `always_enabled` suffix 关键字 |
|------|------------------------------|-------------------------------|
| Interface method 声明 | 合法 | 合法 |
| Module 内 method 实现 | **P0022** | 合法 |
| BVI import method | 合法 | 不适用 |

## 解决方案

**Interface 声明用 pragma，module 实现用 suffix：**

```bsv
// Interface 声明（pragma 形式）
(* always_enabled *)
method Action uart_rx_pin(Bit#(1) val);

// Module 内实现（suffix 形式）
method Action uart_rx_pin(Bit#(1) val) always_enabled;
    uart_rx.uart_rx_in(val);
endmethod
```

P0022 的错误消息只说 "Attributes not allowed for method"，没有提示应该用 suffix 关键字——这导致不熟悉 BSV 语法细节的开发者非常困惑。

## 规则

> Module 内 method 实现不能用 `(* attr *)` pragma 形式，必须用 suffix 关键字（如 `method Action foo() always_enabled;`）。Interface 声明和 BVI import 中两种形式都可以。
