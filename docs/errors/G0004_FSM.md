# G0004_FSM — FSM 中多子模块方法并行调用冲突

## 现象

```
Error: "SdCtrl.bsv", line 100, column 8: (G0004)
  Rule `do_fsm' uses methods that conflict in parallel:
    spi.start(...)
  and
    cmd.build(...)
  and
    resp.select(...)
```

## 原因

`G0004_FSM` 是 `G0004` 的一个特定子情况：在同一 rule 内调用了多个**不同子模块**的 Action 方法（如 `spi.start()` + `cmd.build()` + `resp.select()`）。BSC 将 rule 体内的所有方法调用视为并行执行，因此即使这些方法操作的是不同子模块的内部寄存器，编译器仍然将它们标记为并行冲突。

**关键认知**：
- `descending_urgency` 和 `mutually_exclusive` **无法解决 rule 内部的 G0004**
- 这不是 BSC 的 bug——BSC 保守地假设所有子模块方法可能访问共享资源

## 解决方案

拆为独立规则，每个规则只调用一个子模块的 Action 方法。用 state 寄存器做阶段桥接。

```bsv
// 错误 — 一条 rule 调了 spi + cmd + resp 三个子模块
rule do_fsm (state == ST_INIT && spi.busy == 0);
    spi.start(val);          // spi 子模块
    cmd.build(idx, arg);     // cmd 子模块
    resp.select(T_R1);       // resp 子模块
    state <= ST_NEXT;
endrule

// 正确 — 拆为三个独立 state，每个 rule 只调一个子模块
rule do_init_spi (state == ST_INIT && spi.busy == 0);
    spi.start(val);
    state <= ST_INIT_CMD;
endrule
rule do_init_cmd (state == ST_INIT_CMD);
    cmd.build(idx, arg);
    state <= ST_INIT_RESP;
endrule
rule do_init_resp (state == ST_INIT_RESP);
    resp.select(T_R1);
    state <= ST_NEXT;
endrule
```

## 规则

> FSM 中多子模块的操作必须串行化到独立 rule 中。每个 rule 只调用一个子模块的 Action 方法。用 state 寄存器的 wait state 桥接。
