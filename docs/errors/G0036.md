# G0036 — Write-vs-write 执行顺序依赖（信息性警告）

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 1 captures

## 现象

```
Warning: "UartBridge.bsv", line 56, column 8: (G0036)
  Write to register "ldo_req_flag" by method "ldo_ack" and rule "process_frame"
  have a scheduling dependency, the write by "ldo_ack" is executed before the
  write by "process_frame".
```

## 原因

两个执行主体（method 和 rule，或两个 rule）在同一时钟周期内写同一个寄存器。编译器已按 `descending_urgency` 中指定的顺序正确排除了冲突，G0036 作为 **信息性警告**告知设计者存在这个执行顺序依赖。

对于跨 `(* synthesize *)` 边界的 method-vs-rule write 冲突，G0036 可能无法通过注解消除——它的本质是确认设计者知晓这个执行顺序。

## 解决方案

**审查设计确认执行顺序正确，然后接受此警告。**

如果该警告确实不需要关注：
- 确认 `descending_urgency` 中指定的顺序符合设计意图
- 确认两个执行主体的写操作不会产生竞态条件

如果不希望看到此信息性警告且设计允许：
- 将冲突的寄存器拆分为两个独立寄存器（如 `ldo_req_set` 和 `ldo_req_clear`），各自由不同的执行主体独占写入
- 或使用 `mkWire` 而非 `mkReg` 如果不需要保持状态

## 规则

> G0036 是信息性警告，不表示设计错误。它的作用是告知设计者存在 write-vs-write 的执行顺序依赖，由 `descending_urgency` 解决。审查确认后可以接受。如需消除，拆分寄存器或改用 `mkWire`。
