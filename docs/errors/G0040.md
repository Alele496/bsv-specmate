# G0040 — Urgency 声明中规则自引用

## 现象

```
Error: "Top.bsv", line 28, column 5: (G0040)
  Rule "rl_start" is referenced more than once in the same urgency declaration.
  A rule cannot be both more urgent and less urgent than itself.
```

## 原因

在 `descending_urgency`、`execution_order` 或 `preempts` 属性声明中，同一规则名称在同一个逗号分隔的列表中出现了两次。"rl_a, rl_b, rl_a" 等价于 rl_a > rl_b 且 rl_b > rl_a——编译器检测到 rl_a 的优先级自环，报 G0040。

常见原因：
- 复制粘贴后漏删旧条目
- 长规则列表中人眼查重遗漏
- 宏展开导致规则名重复

## 解决方案

**检查 urgency 声明中每个规则名是否唯一，删除重复项。**

```bsv
// 错误 — rl_start 在同一声明中出现两次
(* descending_urgency = "rl_start, rl_check, rl_done, rl_start" *)

// 正确 — 每个规则只出现一次
(* descending_urgency = "rl_start, rl_check, rl_done" *)
```

## 规则

> 同一 urgency 声明中每个规则名必须唯一。G0040 通常是最容易修复的调度错误——找到重复的规则名，删掉重复的那个。
