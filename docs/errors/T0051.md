# T0051 — Literal 超出寄存器位宽 (×2)

> 适用版本: BSC 2025.07 | 自动生成: 2026-07-15 | 来源: 2 captures

**bsc 输出**：
```
#1: line 104 — Literal 4 is not a valid Bit#(2).
#2: line 171 — Expected type: Bit#(2), Inferred: Bit#(3)
```

**原因**：
1. 寄存器位宽不足以存目标值（如 `Bit#(2)` 需存值 `4`，至少 3 位）
2. 扩大某寄存器位宽后，与之比较/运算的其他寄存器位宽未同步调整

**解决**：

追踪所有相关寄存器的位宽，一并扩宽。

```bsv
// 错误
Reg#(Bit#(2)) ch_seq_idx <- mkReg(0);
Reg#(Bit#(2)) ch_seq_cnt <- mkReg(0);
// ch_seq_cnt 需要存 4 → Bit#(2) 不够

// 正确
Reg#(Bit#(3)) ch_seq_idx <- mkReg(0);
Reg#(Bit#(3)) ch_seq_cnt <- mkReg(0);
```

> **规则**: 扩大寄存器位宽时，所有与之比较/运算的相关寄存器一起扩宽。

> 拼接导致的位置不匹配 → 见 T0060。
