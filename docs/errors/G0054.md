# G0054 — Urgency 引用不存在的规则

## 现象

```
Error: "Top.bsv", line 15, column 5: (G0054)
  Attribute `descending_urgency' references rule "rl_send_data" which does not exist in this module.
```

## 原因

`descending_urgency`、`execution_order`、`preempts` 或 `mutually_exclusive` 等调度属性中引用的规则名在**当前模块**中不存在（未定义或定义在别的模块中）。

常见原因：
- 规则名拼写错误（如 `rl_sendData` 写成 `rl_send_data`）
- 规则已重命名但属性声明未同步更新
- 从其他模块复制属性声明忘记修改规则名
- 规则名含特殊字符（下划线、大小写）写错

## 解决方案

**确认规则名拼写与当前模块中的 rule 声明完全一致（BSV 大小写敏感）。**

```bsv
// 错误 — rl_tx_sender 不存在
(* descending_urgency = "rl_tx_sender, rl_tx_shift" *)
rule rl_tx_shift;
    ...
endrule

// 正确 — 使用实际存在的规则名
(* descending_urgency = "rl_tx_shifter, rl_tx_shift" *)
rule rl_tx_shifter;
    ...
endrule
rule rl_tx_shift;
    ...
endrule
```

## 规则

> urgency 属性只能引用当前模块内定义的规则名。检查拼写和大小写——BSV 中规则名是大小写敏感的。
