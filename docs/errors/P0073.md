# P0073 — 重复声明（类型参数/函数参数/接口方法） (x3)

## 子情况 1：类型参数重复

**bsc 输出**：
```
Error: "MyMod.bsv", line 5, column 8: (P0073)
  Type parameter "t" is declared more than once in the module signature.
```

**原因**：模块的类型参数列表中 `type t` 重复定义。

**解决**：每个 type 参数只声明一次。

```bsv
// 错误
module mkCache#(type t, type depth, type t)(CacheIfc#(t, depth));
//                               ~~~~~~~ t 重复

// 正确
module mkCache#(type t, type depth)(CacheIfc#(t, depth));
```

---

## 子情况 2：函数参数重复

**bsc 输出**：
```
Error: "MyMod.bsv", line 42, column 12: (P0073)
  Function parameter "addr" is declared more than once.
```

**原因**：函数签名中同一个参数名出现两次。

**解决**：重命名其中一个参数。

```bsv
// 错误
function Bit#(32) read(Bit#(8) addr, Bit#(32) addr);
//                                              ~~~~ addr 重复

// 正确
function Bit#(32) read(Bit#(8) addr, Bit#(32) data);
```

---

## 子情况 3：接口方法重复

**bsc 输出**：
```
Error: "MyMod.bsv", line 18, column 10: (P0073)
  Method "read" is declared more than once in this interface.
```

**原因**：同一个 interface 块中声明了两个同名 method。

**解决**：删除重复的 method 声明。

```bsv
// 错误
interface MemIfc;
    method Action write(Bit#(8) addr, Bit#(32) data);
    method Bit#(32) read(Bit#(8) addr);
    method Bit#(32) read(Bit#(8) addr, Bit#(1) burst);  // ← read 重复
endinterface

// 正确 — 给不同读模式不同名字
interface MemIfc;
    method Action write(Bit#(8) addr, Bit#(32) data);
    method Bit#(32) read(Bit#(8) addr);
    method Bit#(32) burst_read(Bit#(8) addr, Bit#(1) burst);
endinterface
```

## 规则

> 类型参数、函数参数、接口方法——每一类中名称必须唯一。BSV 编译器不允许重名，P0073 检测到重复声明即拒绝编译。
