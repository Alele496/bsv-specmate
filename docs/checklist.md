# 编译前检查清单

> 每次编写 BSV 代码后、编译前过一遍此清单。可避免大部分常见编译错误。

## 结构

- [ ] **所有 method 在 rule 之后？** (P0032)
  - 模块内 rule 不能出现在 method 之后
  - 必须 `规则 → method (含 value method)`

## 标识符

- [ ] **没有用 SV 保留字做标识符？** (P0005)
  - 检查：`action` `bit` `byte` `reg` `wire` `module` 等
  - Package 常量用驼峰而非大写

## 类型

- [ ] **Bool 没有用位操作符 `~` `&` `|` `^`？** (T0061)
  - Bool → `!` `&&` `||`；Bit#(n) → `~` `&` `|` `^`

- [ ] **拼接 `{...}` 总位宽 = 目标位宽？** (T0060)
  - 逐一核对拼接各元素位宽之和

- [ ] **字面量不超出目标位宽？** (T0051)
  - `Bit#(2)` 不能赋值为 `4`（需要 3 位）

- [ ] **类型关联的寄存器位宽一致？** (T0051)
  - 修改某寄存器位宽时，与之比较/运算的相关寄存器一并调整

## Rule 调度

- [ ] **每个 rule 内同一寄存器只写一次？** (G0004)
  - `case` 的 `default` 分支不允许写入与其他分支相同的寄存器
  - 显式枚举分支不需要 default

- [ ] **跨 rule 数据传递用了 FIFOF？** (G0010)
  - 不用 `PulseWire + Reg` 组合

- [ ] **跨模块 rule/method 互斥有 urgency 标注？** (G0010)
  - `(* descending_urgency = "rule_a, rule_b" *)`
