# specmate 协作开发流程

> specmate 的三个层次：流程编排 > 项目记忆 > 领域知识。本文档定义第一层——什么时候做什么。

## 标准七步流程

```
第一步: 编码前分析
  specmate_guide(phase="pre_code", input="<任务描述>")
  → 陷阱预警、编码记忆、推荐参考

第二步: 设计范式（涉及已知模式时）
  specmate_guide(phase="pattern", input="<范式名>")
  → 代码骨架 + 变体选择 + 常见陷阱

第三步: 架构决策（有二选一或多选时）
  specmate_guide(phase="decide", input="<选项A vs 选项B>")
  → 对比分析 + 推荐方案

第四步: 写代码
  基于前三步的分析结果编写 .bsv 文件

第五步: 编译前预检
  check_style(files=["bsv/模块名.bsv"])
  → 修复静态检查发现的问题

第六步: 编译
  bsc -u -verilog -g 模块名 文件名.bsv
  通过 → 完成模块
  报错 → 进入第七步

第七步: 报错修复
  specmate_guide(phase="on_error", input="<bsc 完整报错>")
  → 匹配错误码 → 获取原因+方案 → 修复 → 回到第六步

继续下一模块 → 回到第一步
  specmate_guide(phase="continue", input="<下一步描述>")
```

## 关键原则

- **不是出了错才用 specmate，是写代码前就用**。第一步和第五步是最容易被跳过的，也是最关键的。
- **specmate 是搭档，不是外挂**。它的价值不在报错后修得快，而在写代码时帮你少踩坑。
- **设计决策不要自己猜**。第二步和第三步存在就是为了让你在编码前想清楚架构。

## 与 VISION 的对应

| VISION 层 | 流程体现 |
|-----------|---------|
| 流程编排 | 第一~七步的顺序不可颠倒 |
| 项目记忆 | 第六步报错 → 第七步 fix → learn 入库 → 下次同错秒出方案 |
| 领域知识 | 第一步（陷阱）、第二步（模式）、第五步（预检规则）、第七步（错误库） |
